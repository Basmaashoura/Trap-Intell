## ?? Permission System Design

### Type-Safe Constants (no magic strings)
```csharp
Permissions.Honeypots.View    // "honeypots:view"
Permissions.Users.ManageRoles // "users:manage-roles"
Permissions.System.ManagePlans // "system:manage-plans"
```

### 9 Permission Categories, 40+ Permissions
| Category | Permissions |
|----------|------------|
| Honeypots | view, create, update, delete, deploy, configure |
| Attacks | view, analyze, export |
| Alerts | view, acknowledge, escalate, configure |
| Threats | view, analyze, update |
| Reports | view, create, export |
| Dashboards | view, create, manage |
| Users | view, create, update, delete, manage-roles, invite |
| Organization | view, update, manage-settings, manage-billing, manage-api-keys, manage-webhooks |
| System | manage-global-settings, manage-organizations, manage-plans, view-audit-logs, manage-super-admins |
| Commands | view, execute, approve |

### Role Hierarchy
```
SuperAdmin (100) ??? ALL permissions
     ?
OrganizationAdmin (80) ??? Everything except System.*
     ?
SecurityAnalyst (60) ??? Honeypots.*, Attacks.*, Alerts (no configure), Threats.*, Reports.*, Commands (no approve)
     ?
OperationsAnalyst (40) ??? View-heavy + Reports.Create + Alerts.Acknowledge
     ?
Viewer (20) ??? View-only across all domains
     ?
Guest (10) ??? Dashboards.View only
```

## ?? Admin API Endpoints

| Method | Path | Policy | Description |
|--------|------|--------|-------------|
| `GET` | `/api/admin/users` | `users:view` | List organization users |
